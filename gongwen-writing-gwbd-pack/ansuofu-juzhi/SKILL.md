# 安索夫矩阵举措法

**R（Reading）— 原文引用**
> 市场渗透（深耕现有）、市场开发（拓展范围）、产品开发（创新服务）、多元化（开创新局）

**I（Interpretation）— 理解重写**
安索夫矩阵从商业战略引入公文写作，实现工作举措的系统分解。

产品=工作或服务（政策/活动/机制/解决方案）
市场=工作范围或对象（实施领域/覆盖区域/目标群体）

四类举措：
1. 市场渗透（旧产品+旧市场）：在现有职责内优化流程、加强执行、扩大覆盖——核心是做深做实提效
2. 市场开发（旧产品+新市场）：将成熟模式推广到新地区、新渠道、新群体——核心是推广复制
3. 产品开发（新产品+旧市场）：为老对象老领域设计新的服务项目、政策工具——核心是迭代升级
4. 多元化（新产品+新市场）：探索全新工作领域、打造颠覆性服务模式——核心是跨界融合突破

**A1（past case）**
城市园林绿化工作的安索夫矩阵分解：
- 市场渗透（旧产品+旧市场）：深化精细化管养，巩固绿化存量基底
- 市场开发（旧产品+新市场）：推动绿化支补融合，向社区深处实现空间共享、功能叠加
- 产品开发（新产品+旧市场）：实施公园加功能提升，推动绿地从生态景观向城市客厅升级
- 多元化（新产品+新市场）：探索品牌共创与生态价值转化，推动绿化成果转化为城市品牌资产

**A2（future trigger）**
你要谋划"养老服务提升"的工作举措，可以用安索夫矩阵分解。

**E（Execution）— 执行步骤**
1. **明确产品与市场**：产品=养老服务，市场=服务对象/覆盖区域
2. **梳理现有产品和市场**：常规公园绿地/行道树/基础绿化
3. **探索新产品和新市场**：复合功能公园/老旧小区/特定群体需求
4. **两两组合推导出四类举措**
5. **用简洁有力的短语拟标题**

**B（Boundary）— 边界与局限**
- **失效场景**：单一工作不需要强求矩阵分解
- **常见错误**：四类举措混为一谈，没有清晰区分
- **作者盲点**：矩阵是思考工具，不是强制分类
