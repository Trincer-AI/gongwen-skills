# 述职故事化三层面法

**R（Reading）— 原文引用**
> 故事化的本质是以我为主，突出主体性

**I（Interpretation）— 理解重写**
述职故事化三层面（解决"路人甲"问题的核心方法）：

**一、独立负责与深度参与为主（细科优势做法）**
- 不满足于常规操作，运用专业知识进行深度剖析
- 主动引入现代化工具与新方法提升效能
- 凭借高强度精力与贴近一线完成饱和式工作

**二、独立应对与深度思考为主（细科问题重构）**
- 从执行指令到构建机制：不只做完一件事，而是解决一类问题的根源
- 从管理对象到服务协同：不只管住，而是帮助对方达成目标
- 从解决当前到预防一类：不只解决个案，而是建立预防同类问题的标准

**三、价值创造与模式探索为主（细科贡献升华）**
- 从完成项目到创造模式
- 从坚守底线到定义标准
- 从信息中介到知识枢纽

**A1（past case）**
"完善贷款评审体系"的问题重构：
- 【错误】"我配合参与了完善贷款评审体系的工作"
- 【正确】"我针对贷款评审中的人情贷问题，设计并优化打分卡与利率定价模型，推动评审走向标准透明"

**A2（future trigger）**
你在述职中想写"参与了营商环境优化工作"，但这样写你就是路人甲。需要用故事化方法改写。

**E（Execution）— 执行步骤**
1. **聚焦最小单元**：不是写"配合参与了XX工作"，而是写"我独立负责/主导完成的XX最小单元"
2. **突出问题重构**：不写"完成了任务"，而是写"我重构了这个问题"
3. **展示价值创造**：不写"传递了信息"，而是写"我整合创造了什么增量价值"
4. **做到三个突出**：突出专业前沿知识 / 突出现代化工具方法 / 突出高强度饱和覆盖

**B（Boundary）— 边界与局限**
- **失效场景**：确实只是配合参与的工作，强求独立负责反而失实
- **常见错误**：故事化不是写流水账，是提炼有意义的独立单元
- **作者盲点**：有时候团队协作是必要的，但要在协作中明确个人角色
