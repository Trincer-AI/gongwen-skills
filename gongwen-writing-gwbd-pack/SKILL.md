# 公文摆渡技能包

## 目标

为公文写作场景提供一套完整、可执行的技能体系，支持从"写不出来"到"写出亮点"的全链路。

核心解决以下问题：
- 不知道用什么语气写（场景化问题）
- 不知道如何组织结构（框架问题）
- 不知道如何挖掘亮点（深化问题）
- 不知道如何用词精准（语言问题）
- 不知道格式规范（合规问题）

## 何时使用

- "领导让我写个汇报，不知道怎么下笔"
- "写完了领导说太虚、不接地气"
- "工作做了很多但总结写不出来亮点"
- "不知道用口语还是书面语"
- "写材料时被说格式不规范"
- "需要写致辞、讲话稿、述职报告等特定文种"

## 核心职责

本技能包负责将用户的写作需求（模糊的、初步的、有线索的）转化为高质量的公文文本，涵盖：
1. 需求分析：判断任务类型、场景、受众
2. 框架设计：确定结构、逻辑、层次
3. 内容生成：起草、扩写、填充
4. 语言优化：用词、句式、音律
5. 格式校验：规范性、完整性检查

## 路由原则

面对用户的写作需求，按以下维度依次判断：

### 步骤1：判断任务类型

| 用户需求关键词 | 推荐子技能类型 |
|---|---|
| "不知道怎么下笔"/"太虚"/"不接地气" | 场景化写作 (changjing-xiezuo) |
| "不知道用什么语气" | 口语书面语选择 (yuti-xuanze) |
| "不知道结构怎么搭" | 规划类 → 框架类技能 |
| "做了很多但写不出亮点" | 生成类 → 亮点挖掘 (sigong-liangan-faxin) |
| "写完了觉得不够亮" | 优化类 → 亮点深化 (liangdian-chengxu) |
| "不知道用词准不准" | 优化类 → 用词规范 (zhanwei-zhunque-yongci) |
| "格式有没有问题" | 校验类 → 格式规范 (gongwen-guifan) |
| "不知道选什么文种" | 分析类 → 公文三钥匙 (gongwen-sanyaoshi) |

### 步骤2：判断场景约束

| 场景 | 推荐子技能 |
|---|---|
| 表彰大会讲话 | changjing-xiezuo + dairu-shixiang-siwei |
| 检讨/反思发言 | wenti-daoxiang-siwei + huibao-changjinghua |
| 年终述职 | shuzhi-jiegouhua + shuzhi-gushihua |
| 工作汇报 | gongzuo-zongjie-kuangjia + jiaodian-qianzhi |
| 信息报送 | xinxi-baosong-liu-tiaolu + wuzheng-xinxi-xuanti |
| 调研报告 | diaoyan-xinxin + xitong-fenxi |

### 步骤3：判断技能串行依赖

- **先分析后生成**：拿到需求先判断场景 (changjing-xiezuo) → 再选框架 (gongzuo-zongjie-kuangjia/jiaodian-qianzhi) → 再起草
- **先生成后优化**：初稿 (changjing-xiezuo/dairu-shixiang-siwei) → 亮点挖掘 (sigong-liangan-faxin) → 语言优化 (xiaoci-dianjing/yinyv-zhuangzhonggan) → 格式校验 (gongwen-guifan)
- **先框架后填充**：结构设计 (tongxin-yuan-lunzheng/zongfen-pingxing-tuijin) → 内容填充 (san-ji-lingdao-jianghua-luoji/houban-juhua-luoji) → 承启句 (chengqi-ju-xiezuo)

## 失败与回退策略

### 链路失败时的回退

| 失败节点 | 回退策略 |
|---|---|
| changjing-xiezuo 后不知如何组织 | 回退到 mubiao-daoxiang-yu-jieguo-daoxiang 判断组织方式 |
| 框架搭好后内容空洞 | 回退到 sigong-liangan-faxin 重新挖掘亮点 |
| 语言写出后不够庄重 | 回退到 yinyv-zhuangzhonggan 重新调整音律 |
| 格式校验发现问题 | 回退到对应文种的生成类技能重新起草 |

### 技能冲突时的判断

| 冲突场景 | 判断原则 |
|---|---|
| changjing-xiezuo vs gongwen-guifan | 场景化优先于格式（先对后准） |
| 亮点挖掘 vs 格式规范 | 亮点优先（无亮点则无意义） |
| 多种框架可选 | jiaodian-qianzhi（前置）优先于 jiaodian-houzhi（后置），除非用户明确要求"以成效为焦点" |
| 口语vs书面语冲突 | 讲话稿选口语，政策文件选书面，汇报稿可混合 |

## 示例

### 示例1：用户说"领导让我写个工作汇报，不知道怎么下笔"

1. **激活**：changjing-xiezuo（场景化写作）作为入口skill

3. **框架**：gongzuo-zongjie-kuangjia 或 jiaodian-qianzhi
4. **亮点**：sigong-liangan-faxin
5. **优化**：xiaoci-dianjing + yinyv-zhuangzhonggan
6. **校验**：gongwen-guifan
