# 工作总结焦点前置框架法

**R（Reading）— 原文引用**
> 焦点前置：从工作本身出发，以主线为焦点，围绕五个维度拆解工作。

**I（Interpretation）— 理解重写**
焦点前置框架是一种从工作本身出发的框架搭建方法。

核心是找到工作的"主线"，以主线为核心组织全部材料。标题采用两段式，前段是主线。

五维度拆解：
1. **出发点** — 为什么做这件事（工作的依据和背景）
2. **主线** — 核心工作是什么（工作的主要内容）
3. **手段** — 用什么方式方法（工作的方法路径）
4. **成效标准** — 衡量成果的标尺（工作的目标和检验标准）
5. **保障** — 需要什么支撑（工作的条件和资源）

这套框架适合：工作有明确主线可以拎起来、需要展现工作的系统性、领导关注"怎么干"的逻辑。

**A1（past case）**
某市城管局年度工作总结：

**旧框架**（按事项分）：
- 队伍建设
- 业务工作
- 宣传工作
- 基础设施建设

**新框架**（焦点前置）：
```
一、坚持人民至上，夯实精细化管理出发点
二、聚焦科技赋能，扭住精细化管理主线
三、突出制度创新，用好精细化管理手段
四、对标群众满意，明确精细化管理成效标准
五、强化队伍建设，筑牢精细化管理保障
```

效果：领导一眼看出"精细化管理"是核心，五个维度围绕这一主线展开，结构清晰、逻辑统一。

**A2（future trigger）**
你要写一份某单位"优化营商环境"年度工作总结，工作中涉及多个方面：制度建设、流程再造、队伍建设、信息化建设、监督检查。如何用焦点前置框架组织这份总结？

**E（Execution）— 执行步骤**
1. **找主线**：从多项工作中提炼出核心主线（如"优化营商环境"）
2. **定维度**：围绕主线设计五维度（出发点/主线/手段/成效标准/保障）
3. **填内容**：把具体工作填入对应维度
4. **调标题**：标题采用两段式，前段是主线，后段是维度关键词

**B（Boundary）— 边界与局限**
- **失效场景**：
  - 工作没有明确主线、各项工作的逻辑关联不强（强行找主线会扭曲工作本身）
  - 创意性/艺术性总结（程式化框架会束缚表达）
  - 短期简单任务（周报、日报不需要完整框架）
- **常见错误**：
  - 主线找偏：五维度不能围绕主线展开（自圆其说式的强行找主线）
  - 维度不统一：同一层级用了不同分类标准（前段用事项、后段用主体）
  - 框架僵化：所有工作都用同一主线（"党建引领"成了万能主线）
  - 标题工整但内容空洞（框架好但填充内容跟不上）
- **作者盲点**：
  - 过于追求框架的完美，忽视了内容的实质性
  - 框架是工具，不是目的——内容为王
  - 焦点前置适合展示工作逻辑，但可能掩盖工作亮点（成效被框架压缩）
