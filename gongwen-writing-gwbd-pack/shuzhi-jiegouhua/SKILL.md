# 述职结构化三化法

**R（Reading）— 原文引用**
> 将复杂信息压缩成易于记忆和传播的符号。结构化将复杂信息压缩成易于记忆和传播的符号，在青年干部的述职发言中主要应用在三个层面：标签化、数据化、框架化

**I（Interpretation）— 理解重写**
述职结构化三层次：

**一、标签化（用公式提炼个人品牌）**
- 数字概括式：数词+核心词 → "在三根线头中织就青春答卷"
- 角色定义式：对象+角色 → "当好水管家"
- 战略升维式：从A到B → "从投项目到预生态"

**二、数据化（用数据群构建论证）**
- 维度覆盖法：同一任务不同阶段的关键指标组合（证明全流程参与）
- 点面结合法：典型案例数+整体覆盖数（展示精度与广度）
- 价值锚定法：数据与岗位核心价值直接挂钩（工作价值最有力宣告）

**三、框架化（用模型组织述职内容）**
- 综合管理服务支撑类岗位：按价值维度结构化（放弃办文办会办事的流程罗列）
- 专业技术业务执行类岗位：按职能模块结构化（遵循本职核心板块）

**A1（past case）**
"283个共排水方案审查、174次闭水试验、指导175个项目核查"的结构化逻辑：
- 283个方案审查 = 前端规划设计阶段把关（专业判断力）
- 174次闭水试验指导 = 终端施工建设阶段监督（深入一线解决问题）
- 175个项目核查 = 后端成果验收阶段检验（负责到底的严谨性）

**A2（future trigger）**
你是某园区管理委员会的青年干部，负责管网建设维护工作，需要在述职中展示你的工作量和工作能力。

**E（Execution）— 执行步骤**
1. **标签化**：提炼一句高度概括的个人品牌（数词+核心词 / 对象+角色 / 从A到B）
2. **数据化**：用维度覆盖法/点面结合法/价值锚定法构建数据论证
3. **框架化**：按价值维度或职能模块组织述职内容
4. **避免两个误区**：数据堆砌无逻辑关联 / 框架罗列太琐碎

**B（Boundary）— 边界与局限**
- **失效场景**：数据不够多时强求数据化反而显得单薄
- **常见错误**：数据没有结构化设计，只是简单罗列；标签太普通没有辨识度
- **作者盲点**：述职时间有限，不是数据越多越好，要精选有论证力的数据
