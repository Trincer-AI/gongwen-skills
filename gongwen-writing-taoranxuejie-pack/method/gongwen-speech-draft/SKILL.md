# 领导讲话稿撰写

## R — 原文

"在动笔之前，要按照事物从无到有的逻辑顺序，问自己如下几个问题。1.谁来讲话？讲给谁听？2.会议背景是什么？3.领导想讲什么？"

## I — 方法论骨架

**讲话稿撰写拆解四步** = 明确讲话者/受众 → 了解会议背景 → 分析构成要素 → 搭框架

核心原则：连环追问挖出真实背景，避免只问一次

结构框架：讲形势→讲问题→讲对策→讲要求

## A1 — 书中的应用

四步操作详解：
1. 明确讲话者/受众：通过追问确认层级和范围（谁来讲？讲给谁？）
2. 了解会议背景：前世今生、真正目的、上级是否点过本单位
3. 分析构成要素：帽段/意义/问题/对策/要求
4. 搭框架：按讲形势→讲问题→讲对策→讲要求结构

连环追问技巧：不要只问一次，要连环追问挖出真实背景

## A2 — 触发场景

**秘书场景**："领导下午有个会，帮我写份讲话稿"

**思维顾问场景**："讲话稿的撰写和普通公文有什么区别？重点在哪里？"

**角色扮演场景**："假设你是领导，列出你希望在讲话稿中看到的要点"

## E — 可执行步骤

### Step 0 - 预检

- [ ] 确认是讲话稿（用于会议发言），不是其他文种
- [ ] 确认是否有情感张力要求（演讲稿/宣讲稿需要一气呵成）
- [ ] 确认时间是否充裕（讲话稿需要深度沟通）

### Step 1 - 明确讲话者与受众

- [ ] 追问：谁来讲话？什么级别的领导？
- [ ] 追问：讲给谁听？受众群体是什么？
- [ ] 确认：领导风格偏好（简洁/详细/数据/情怀）

### Step 2 - 挖掘会议背景

- [ ] 追问：会议背景是什么？前世今生如何？
- [ ] 追问：会议真正目的是什么？
- [ ] 追问：上级是否点过本单位？有什么特殊要求？
- [ ] 连环追问：不要只问一次，挖出真实背景

### Step 3 - 分析构成要素

- [ ] 帽段：开场、背景介绍、出席人员
- [ ] 意义：为什么要开这个会/做这项工作
- [ ] 问题：当前存在的主要问题
- [ ] 对策：解决问题的措施和方法
- [ ] 要求：对下一步工作的要求

### Step 4 - 搭建框架

- [ ] 按讲形势→讲问题→讲对策→讲要求结构
- [ ] 先与领导确认框架再动笔
- [ ] 避免闷头写错过沟通时机

### Step 5 - 撰写与调整

- [ ] 动笔撰写，完成初稿
- [ ] 使用换位技巧调整站位
- [ ] 进行复核检查

## B — 边界与Fallback

### 边界条件表

| 边界类型 | 触发条件 | 处理动作 |
|----------|----------|----------|
| 场景边界 | 情感类致辞/演讲稿 | 不适用此框架，需用演讲稿专属写法 |
| 沟通边界 | 领导不配合沟通背景 | 查阅该领导历史讲话稿推断风格 |
| 风格边界 | 不同领导风格差异大 | 个性化调整，不能套用统一模板 |
| 时间边界 | 时间紧迫无法深度沟通 | 使用典型结构模板先搭框架再细化 |
| 站位边界 | 讲话稿站位不能超越领导权限 | 按 fw-004 换位技巧先确认站位 |

### Fallback策略表

| 困境 | 应对策略 | 操作要点 |
|------|----------|----------|
| 领导不配合沟通 | 查阅历史讲话稿 | 目标≥2篇，分析语言风格和常用词 |
| 时间紧迫（<2小时） | 使用典型结构模板 | 帽段+意义+问题+对策+要求，直接填充 |
| 连环追问被敷衍 | 限定追问次数（≤3次） | 第3次追问后取最后一次回答为基准 |
| 不确定领导意图 | 保守写法 | 使用"讲形势→讲问题→讲对策→讲要求"常规套路 |
| 担心站错位 | 先确认站位 | 按 fw-004 与领导确认大方向再动笔 |

**连环追问规范**：追问次数限定≤3次，第3次后取最后回答为基准，避免无限追问延误时效。
