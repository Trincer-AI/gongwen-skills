## R - 原文定义

> 在《从零开始学公文写作》中，我详细介绍了职场新人建立素材'兵器库'的步骤和方法，比如使用site指令实现精准检索；使用微信检索功能检索政务号文章；联系纵向垂直关系的上级部门、沟通横向平行关系的兄弟单位、深挖本单位的宝藏资料等。

**素材兵器库**是指执笔人积累的各类公文素材资源库，包括：检索方法库（site指令、政务号检索等）、渠道库（上下级部门、兄弟单位等）、范本库（优秀公文、框架模板等）、金句库（好词好句、标题结构等）。建立完善的'兵器库'是公文写作的基础，需要日常持续积累和维护。

## I - 自己的解释

**公文格式基础知识**是公文写作的"兵器库"基础——知道有什么武器、武器在哪里、怎么用。

公文格式不是孤立存在的，它由**四大要素**构成：
1. **文种选择**——用什么文种（通知/报告/函...）
2. **格式规范**——标题/主送/正文/落款的位置和字体要求
3. **内容结构**——不同文种的框架和层次安排
4. **语言风格**——正式程度、站位高低、用词规范

这四者相互配合：选对文种才有合适的格式依据，有清晰的结构才能组织内容，语言风格则要匹配讲话人的身份和场合。

## A1 - 书中的案例

**通知写作的全景回顾法**（fw-010）：
把自己想象为参会人员，预演从出门到抵达会场的每一个细节。公式：通知要素全景 = 时间 + 地点 + 任务 + 准备事项 + 着装要求 + 车证管理 + 联系人 + 应急预案。

**通讯稿的本地优先原则**（pr-013）：
本单位已经刊发的稿件是经过领导审阅认可的，是最好的参照，不要舍近求远。这是"兵器库"思想的核心应用——善用手边已有的优质资源。

## A2 - 触发场景

### 场景一：秘书触发
**情境**：领导让你写一份关于安全生产的通知
**你的行动**：先检索本单位历史通知模板，套用格式；再查上级主管部门的最新要求，补充内容；最后检查要素是否齐全（时间/地点/参与人员/具体要求）

### 场景二：思维顾问触发
**情境**：同事问"这份材料的格式有什么问题"
**你的分析**：从四个维度检查——
1. 文种是否匹配工作性质（部署用通知、表彰用通报、咨询用函...）
2. 格式是否规范（标题是否"三要素"、主送机关是否正确）
3. 结构是否完整（帽段→主体→结尾）
4. 语言是否得体（站位是否匹配讲话人身份）

### 场景三：角色扮演触发
**情境**：领导说"这个材料格式不对"
**你的模拟扮演**：
- 作为领导视角：看标题是否简洁明了、主送是否清晰、正文逻辑是否通顺
- 常见领导挑剔点：格式不庄重（用了黑体过多）、层次不清（序号跳跃或混杂）、落款信息不全
- 建议：先把材料"降维"为自己能理解的形式，检查后再提交

## E - 如何应用（质量检查点模式）

### 步骤一：建立格式清单
**Input**: 本单位常用文种列表
**Output**: 可打印的格式检查卡片

**质量检查点**：
- [ ] 通知格式：标题三要素 + 主送机关 + 正文 + 落款日期
- [ ] 报告格式：标题 + 主送 + 正文(情况/问题/建议) + 落款
- [ ] 函格式：标题 + 主送 + 正文 + 落款
- [ ] 每类文种保留2-3篇标杆范本

### 步骤二：积累范本库
**Input**: 单位历年优秀公文
**Output**: 按文种分类的范本库

**质量检查点**：
- [ ] 每类文种保留2-3篇标杆范文
- [ ] 标注关键结构和常用句式
- [ ] 定期更新，删去过时内容

### 步骤三：检索验证
**Input**: 待写材料的文种和主题
**Output**: 检索到的参考素材

**【用户确认】** 检索完成后，请确认检索到的范本是否符合本单位最新规范：
- [ ] 已查阅上级部门最新政策文件
- [ ] 已查阅本单位历史同类材料
- [ ] 确认范本来源可靠、可直接参照

**质量检查点**：
- [ ] 上级部门最新政策（site:gov.cn）
- [ ] 本单位历史同类材料
- [ ] 平行兄弟单位的参考素材

**检索优先级**：
| 优先级 | 来源 | 说明 |
|-------|-----|-----|
| 1 | 本单位历史范本 | 经过领导审阅，最可借鉴 |
| 2 | 上级部门最新文件 | 政策依据，表达规范 |
| 3 | 平行兄弟单位 | 拓展思路，锦上添花 |

## B - 边界与误区（表格化）

### 适用边界
| 场景 | 是否适用 | 说明 |
|-----|---------|-----|
| 法定公文文种（15种） | 是 | 严格按规范执行 |
| 企业公文/内部管理制度 | 是 | 基本原则适用，格式可简化 |
| 文学性文章/随笔 | 否 | 不在此范畴 |

### 常见错误
| 错误类型 | 具体表现 | 正确做法 |
|---------|---------|---------|
| 文种混用 | 该用"请示"的用了"报告" | 明确：请示需要批复，报告不需要 |
| 格式残缺 | 缺少主送机关或落款日期 | 按规范逐项检查 |
| 语言不匹配 | 普通干部发言用了领导人措辞 | 站位匹配讲话人身份 |
| 格式过繁 | 小通知用了大报告的复杂格式 | 简洁明了，符合文种性质 |
| 舍近求远 | 只靠百度，忽视本单位资源 | 先内部再外部 |

## F - 参考资源

### 格式规范查询
- **国家标准**：GB/T 9704-2012《党政机关公文格式》
- **上级部门**：本单位上级主管部门的格式要求
- **历史范本**：本单位历年审定的标准文件

### 兵器库维护
- 每月整理一次范本库，删旧增新
- 建立电子档案，标注文种/年份/用途
- 重要文件打印存档，双重备份
