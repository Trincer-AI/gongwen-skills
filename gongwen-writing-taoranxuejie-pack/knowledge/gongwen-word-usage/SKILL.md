## R - 原文定义

> 在问题14中，小白的选词、用词问题是'大人物用小词'，把讲话人的站位拉低了。

**"大人物用小词"**是选词不当的一种典型错误——讲话人的身份、职务、场合要求用正式、庄重的表达，但执笔人用了过于口语化、生活化的词汇，把应该高大上的站位拉低了。

> 在工作中，还有一类问题恰恰相反，是'小人物用大词'，选词、用词同样不符合人物的年龄、身份和场景。

**"小人物用大词"**是另一种选词不当——普通干部、基层员工发言，却用了过于宏大、庄重的词汇，显得不真实、不贴切。

## I - 自己的解释

**易错字词辨析**解决的是公文写作中"词不达意"或"词不对位"的问题。选词用词有三条基本准则：

### 准则一：站位匹配
- 领导讲话 → 正式、庄重、有高度
- 同事沟通 → 平等、务实、有建设性
- 基层发言 → 具体、朴实、有真情

### 准则二：场合匹配
- 正式会议 → 严肃、规范、有分寸
- 座谈交流 → 亲切、自然、有互动
- 表彰鼓励 → 热情、有力、有感染力

### 准则三：语境匹配
- 上行文 → 谦逊、尊重、有分寸
- 下行文 → 明确、有力、有权威
- 平行文 → 对等、礼貌、有商榷

## A1 - 书中的案例

**乐高积木模型**（tm-011）：
把每个词汇比作一块乐高积木，熟悉每块积木的样子，就能拼出包装上展示的玩具吗？答案显然是否定的，因为每块积木都有正确的位置，位置不对，再熟悉也没用。

核心观点：
1. 词汇没有对错，只有位置对错
2. 词汇必须在正确位置才有价值
3. 关键是理解词汇的"逻辑结构"——何时用、何地用、怎么用

**长难句崇拜的误区**（ce-005）：
很多人认为优质公文是由长难句组成的。实际恰恰相反，对于千字以内的公文，简单句完全可以满足工作需求，且不容易出错。真正应该追求的不是句子的长度和复杂度，而是句式结构的精准和对仗。

## A2 - 触发场景

### 场景一：秘书触发
**情境**：领导让你写一份表彰先进的通报
**你的行动**：
1. 检查用词是否匹配"表彰"基调（避免口语化）
2. 检查是否做到"大人物用大词"（表彰对象如果是领导，词汇要有高度）
3. 检查是否避免"小人物用大词"（如果是普通员工，不能用过大的词汇）
4. 检查语言是否精准（避免歧义）

### 场景二：思维顾问触发
**情境**：同事说"我写的材料领导说站位不对，哪里有问题"
**你的分析**：
可能的问题类型：
- **"大人物用小词"**：领导讲话用了"搞定""没问题"等口语
- **"小人物用大词"**：普通员工发言用了"务必""必须"等命令式词汇
- **语境错配**：内部讨论用了对外公开发布的正式措辞

建议：
1. 先确认讲话人的身份和场合
2. 再核对用词是否匹配
3. 读出声检查是否有突兀感

### 场景三：角色扮演触发
**情境**：扮演上级领导审阅材料
**你的模拟**：
- 看到"大家要齐心协力把这件事搞定" → 评判：口语化，站位不对
- 看到"必须深入贯彻...的重要思想" → 评判：对普通员工而言语气过重
- 建议：按"谁来讲、讲给谁听、在什么场合"三个要素核对用词

## E - 如何应用（质量检查点模式）

### 步骤一：明确讲话人身份
**Input**: 待写材料的背景信息
**Output**: 确认的站位定位

**质量检查点**：
- [ ] 谁来讲？（讲话人的职务、级别、年龄）
- [ ] 讲给谁听？（受众的职务、级别、与讲话人的关系）
- [ ] 在什么场合？（正式会议/座谈交流/内部讨论）

**站位定位表**：
| 讲话人 | 场合 | 语言风格 |
|-------|-----|---------|
| 领导 | 正式会议 | 正式、庄重、有高度 |
| 同事 | 座谈交流 | 平等、务实、有建设性 |
| 基层 | 基层发言 | 具体、朴实、有真情 |

### 步骤二：核对词汇站位
**Input**: 确认的站位定位
**Output**: 词汇核对结果

**质量检查点**：
- [ ] 没有"大人物用小词"（领导讲话无口语化词汇）
- [ ] 没有"小人物用大词"（普通员工发言无过度庄重词汇）
- [ ] 语境匹配（上行文谦逊、下行文有力、平行文礼貌）

**常见站位问题检测**：
| 场合 | 避免用语 | 推荐用语 |
|------|---------|---------|
| 领导讲话 | 搞定、没问题、小意思 | 务必、着力、切实 |
| 基层发言 | 必须、务必、严肃处理 | 积极、努力、争取 |
| 内部讨论 | 贯彻落实、隆重召开 | 推进、开展、进行 |
| 对外发布 | 我单位、本部门 | XX单位、贵单位 |

### 步骤三：出声朗读检验
**Input**: 写完的正文
**Output**: 朗读检验结果

**【用户确认】** 提交前请确认朗读检验已完成：
- [ ] 朗读时无突兀感或翻译腔
- [ ] 用词站位与讲话人身份匹配
- [ ] 简短精准，非口语化也非过度书面化

**质量检查点**：
- [ ] 这句话读起来是否自然？
- [ ] 是否有"翻译腔"或"官腔"的突兀感？
- [ ] 换一个人讲这句话是否合适？
- [ ] 简短+精准（简短不是口语化）

## B - 边界与误区（表格化）

### 适用边界
| 场景 | 是否适用 | 说明 |
|-----|---------|-----|
| 各类公文语言表达 | 是 | 基本原则适用 |
| 不同地区/单位 | 是 | 结合本单位实际调整 |
| 文学性创作 | 否 | 不在此范畴 |

### 常见错误
| 错误类型 | 具体表现 | 正确做法 |
|---------|---------|---------|
| 大人物用小词 | 领导讲话用口语 | 按领导身份选择正式词汇 |
| 小人物用大词 | 员工发言用命令式 | 语言贴切身份，避免居高临下 |
| 长难句堆砌 | 每句都很长很复杂 | 简句为主，长短搭配 |
| 翻译腔 | 读起来像机器翻译 | 用母语思维，检查是否通顺 |
| 词不达意 | 词汇与语境不匹配 | 按"谁来讲/讲给谁/什么场合"核对 |

### 词汇积累工具
- 建立本单位的"高频词汇表"
- 分类积累：领导讲话词/工作推进词/总结汇报词
- 重要词汇标注使用场景

### 站位匹配检测
- 写完后问自己：这个词汇XX级别的领导会不会用？
- 出声朗读检查是否有突兀感
- 对照"乐高积木"思维：词汇没有对错，只有位置对错
