## R - 原文定义

根据《党政机关公文处理工作条例》，法定公文共**15种**文种，分为：

**上行文**（向上级机关报送）：
1. **请示**——请求指示、批准（需要批复）
2. **报告**——汇报工作、反映情况（不需要批复）
3. **议案**——同级政府或人大提请审议事项

**平行文**（向同级或不相隶属机关）：
4. **函**——询问、商洽、请求批准
5. **平行文意见**——对重要问题的见解和处理办法

**下行文**（向下级机关下发）：
6. **决议**——会议讨论通过的重大决策
7. **决定**——重要事项或重大行动安排
8. **命令/令**——公布行政法规和规章
9. **指示**——向下级布置工作
10. **公报**——公开发布重要决定或重大事件
11. **公告**——向社会公布重要事项
12. **通告**——公布社会各有关方面应遵守的事项
13. **通知**——发布/转发/批转公文，部署工作
14. **通报**——表彰先进、批评错误、传达重要情况
15. **批复**——答复下级机关请示事项

## I - 自己的解释

**15种法定公文文种**是机关公文写作的基础"兵器库"。选对文种是写好公文的第一步。

文种选择的**三层筛选逻辑**：

### 第一层：方向筛选
- 看文件是**往上走、往下走、还是平行走**
- 上行文：请示、报告、议案
- 下行文：决议、决定、命令、指示、公报、公告、通告、通知、通报、批复
- 平行文：函、意见

### 第二层：功能筛选
- **请求事项**→请示（有期批复）/ 函（请求批准）
- **告知情况**→报告（往上）/ 通报（往下）
- **公布事项**→公告（社会）/ 通告（特定范围）
- **部署工作**→通知（往下）/ 指示（往下）

### 第三层：细节筛选
- **要不要批复**→请示要批复，报告不要
- **公开还是内部**→公告向社会，通告限范围
- **重大还是一般**→决定管重大事项，通知管常规工作

## A1 - 书中的案例

**述职报告vs工作总结区分框架**（fw-012）：
述职报告是报告体公文，用于向他人介绍自己的工作情况，因为使用第一人称，总体文风会稍显轻松；工作总结是对工作成果的总结汇报，站位更高，文风更加内敛、严谨、庄重。

两者区别：
| 维度 | 述职报告 | 工作总结 |
|------|---------|---------|
| 人称 | 第一人称 | 第三人称 |
| 文风 | 稍显轻松 | 严肃庄重 |
| 内容 | 德能勤绩廉 | 政治/业务/党建 |

**通知的全景回顾法**（fw-010）：
通知要素全景 = 时间 + 地点 + 任务 + 准备事项 + 着装要求 + 车证管理 + 联系人 + 应急预案

## A2 - 触发场景

### 场景一：秘书触发
**情境**：领导让你处理一项工作汇报
**你的行动**：
1. 判断是给上级汇报（上行文）
2. 看是否需要上级表态或批复
   - 需要批复 → 选**请示**
   - 不需要批复 → 选**报告**
3. 按对应文种的格式规范起草

### 场景二：思维顾问触发
**情境**：同事问"请示和报告有什么区别"
**你的回答**：
- 本质区别：**请示需要批复，报告不需要**
- 表述差异：请示结尾写"请批示"，报告结尾写"特此报告"
- 常见错误：用报告代替请示（导致该批复的事项没批复）

### 场景三：角色扮演触发
**情境**：扮演上级领导审阅材料
**你的模拟**：
- 看到用"报告"处理需要"批复"的事项 → 评判：格式错误，需改为"请示"
- 看到向同级单位发"通知" → 评判：文种方向错误，应改用"函"
- 看到材料格式完整但内容空洞 → 评判：缺乏实质内容，属于"假大空"

## E - 如何应用（质量检查点模式）

### 步骤一：明确行文方向
**Input**: 待处理的工作事项
**Output**: 确定行文方向（上/下/平）

**质量检查点**：
- [ ] 这份文件往哪个方向走？（往上/往下/平行）
- [ ] 我的身份是什么？（发文机关的级别）
- [ ] 对方需要做什么？（批复/知晓/执行）

### 步骤二：匹配文种功能
**Input**: 行文方向
**Output**: 选定的文种

**【用户确认】** 选择文种前，请确认以下判断：
- [ ] 这份文件是往上、往下还是平行？
- [ ] 是否需要上级明确答复/批复？
- [ ] 是告知情况还是部署工作？

**质量检查点**：
- [ ] 往上 + 请求批准 → 请示
- [ ] 往上 + 单纯汇报 → 报告
- [ ] 往下 + 部署工作 → 通知
- [ ] 往下 + 表彰批评 → 通报
- [ ] 平行 + 商洽请求 → 函

**文种选择决策表**：
| 场景 | 文种 | 关键判断 |
|-----|-----|---------|
| 请求批准（需批复） | 请示 | 需要上级明确答复 |
| 请求批准（不相隶属） | 函 | 无隶属关系 |
| 单纯汇报（不需批复） | 报告 | 让上级知晓 |
| 部署工作 | 通知 | 布置任务要求 |
| 表彰/批评/传达 | 通报 | 告之事项 |

### 步骤三：检查格式配套
**Input**: 选定的文种
**Output**: 格式检查结果

**质量检查点**：
- [ ] 标题符合"发文机关+关于+事项+文种"三要素
- [ ] 主送机关正确
- [ ] 结语与文种匹配（"请批示"/"特此报告"/"请予函复"）

## B - 边界与误区（表格化）

### 适用边界
| 场景 | 是否适用 | 说明 |
|-----|---------|-----|
| 党政机关公文 | 是 | 严格按15种法定文种 |
| 事业单位公文 | 是 | 参照执行 |
| 企业内部文书 | 否 | 不属于法定公文，格式可灵活 |

### 常见错误
| 错误类型 | 具体表现 | 正确做法 |
|---------|---------|---------|
| 文种混淆 | 该用"请示"的用了"报告" | 明确是否需要批复 |
| 方向错误 | 给上级发"通知" | 上行文用请示或报告 |
| 语气不当 | 内部事务用"函" | 同一机关内部不用函 |
| 结语缺失 | 请示结尾没有"请批示" | 结语必须与文种匹配 |
| 函的滥用 | 对有隶属关系的上级用函 | 有隶属关系不用函 |
