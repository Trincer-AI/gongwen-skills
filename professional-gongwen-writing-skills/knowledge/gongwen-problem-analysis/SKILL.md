## R - 原文定义

> 思维定式误人——上一项工作影响对下一项工作的审题；熟悉的工作影响对陌生的工作的审题；低难度工作影响对高难度工作的审题。三种心态的本质都是趋利避害——人总是更愿意做自己擅长的事情。

**"四看"审题法**揭示了审题常犯的三种错误：
1. **思维定式**——套用上一工作模板
2. **急于表现**——想快速完成博好感
3. **没搞清领导意图**——闷头写，错过沟通时机

正确做法：接任务后先复述确认理解，主动追问背景（连环问挖出真实原因），避免"闷头写"错过沟通时机。

## I - 自己的解释

**问题分析框架**是面对公文写作任务时的系统思考方法，核心是**先理解再动笔**。

这个框架解决的是"提笔没话说"的根本问题——不是不会写，而是没搞清要写什么。

问题分析的四步法：

### 第一步：确认任务（复述理解）
拿到任务后，先用自己的话复述一遍：
- "您的意思是让我写一份关于……的通知，对吗？"
- "这份材料是给谁看的，需要包含哪些要素？"

### 第二步：追问背景（连环追问）
不要只问一次，要连环追问：
- "这个工作是针对什么问题/情况？"
- "上周/上次有没有类似的工作？"
- "领导想强调什么重点？"
- "有没有需要特别注意的措辞或站位？"

### 第三步：分析要素
明确后，分析材料的基本构成：
- **帽段**：为什么写（背景/意义）
- **主体**：写什么（情况/问题/对策）
- **结尾**：怎么收（要求/呼吁/总结）

### 第四步：核对要求
写完后，对照任务要求逐项检查：
- 是否回应了领导的真实意图？
- 是否有遗漏的要素？
- 站位是否正确？

## A1 - 书中的案例

**脚手架理论**（tm-013）：
写公文前必问自己三个问题：
1. "为什么要写这个呢？"——了解工作背景，找准站位和方向
2. "上周才写过，为什么又要写？"——思考同一工作不同阶段的差异和侧重
3. "领导想说点什么呢？"——努力贴合领导思路

三问定全局，是搭建公文章节框架前的必做功课。

**"以小见大"述职法**（fw-020）：
帽段亮眼（不用感情词，句逗分明，逻辑层次清晰）；工作分类整合（把琐碎工作归类到几个维度）；提炼链条式亮点（把录用、考核、任命提炼为激励机制）。

## A2 - 触发场景

### 场景一：秘书触发
**情境**：领导布置了一项新的工作汇报任务
**你的行动**：
1. 先复述确认："您是说，让我写一份关于上半年工作进展的汇报材料，对吗？"
2. 连环追问："这份材料是向上级汇报还是向同级通报？重点是成效还是问题？有没有特别想强调的工作？"
3. 按"脚手架三问"分析背景
4. 搭建框架后动笔

### 场景二：思维顾问触发
**情境**：同事问"我每次写材料都被领导打回来，说没写到点子上，怎么办"
**你的分析**：
问题可能出在：
- 审题不清——没搞清领导真正想要什么
- 站位不对——写了下属的话或过细的话
- 结构混乱——眉毛胡子一把抓，没有重点

建议：
1. 每次接任务后先复述确认理解
2. 连环追问3个以上问题挖背景
3. 写提纲给领导过目，确认方向后再写

### 场景三：角色扮演触发
**情境**：扮演上级领导审阅材料
**你的模拟**：
- 看到材料内容空洞、套话连篇 → 评判：执笔人对工作理解不透彻，属于"假大空"
- 看到材料细节过细、越俎代庖 → 评判：写了本该是下属写的具体执行内容，站位错误
- 建议：写材料前先按"脚手架三问"过一遍，确保方向正确

## E - 如何应用（质量检查点模式）

### 步骤一：复述确认
**Input**: 领导布置的任务
**Output**: 确认后的任务理解

**质量检查点**：
- [ ] 24小时内完成复述确认
- [ ] 用自己的话复述任务内容
- [ ] 确认发文方向（上行/下行/平行）
- [ ] 确认受众（给谁看，他想看到什么）
- [ ] 确认截止时间（给领导留出修改时间）

### 步骤二：连环追问
**Input**: 初步理解的任务
**Output**: 深化的背景理解

**质量检查点**：
- [ ] 主动问3个以上具体问题
- [ ] "这项工作的背景是什么？"（挖原因）
- [ ] "之前有没有类似的工作？"（找参照）
- [ ] "领导最想强调哪一点？"（抓重点）
- [ ] "有没有措辞上的特别要求？"（避雷区）

**追问深度评估**：
| 层级 | 问题类型 | 示例 |
|-----|---------|-----|
| 第一层 | 表层问题 | "这个材料什么时候要？" |
| 第二层 | 背景问题 | "这个工作的背景是什么？" |
| 第三层 | 关联问题 | "和上次XX工作有什么关联？" |
| 深层 | 意图问题 | "领导最想强调什么？" |

### 步骤三：脚手架搭框架
**Input**: 深化的背景理解
**Output**: 给领导确认的写作提纲

**【用户确认】** 提交给领导确认前，请确认已完成脚手架三问：
- [ ] 为什么要写这个？（背景/意义已清晰）
- [ ] 和之前的工作有什么关系？（差异点已分析）
- [ ] 领导想表达什么？（核心观点已明确）

**质量检查点**：
- [ ] 问自己：为什么要写这个？→ 确定帽段方向
- [ ] 问自己：和之前的工作有什么关系？→ 确定与历史材料的差异
- [ ] 问自己：领导想表达什么？→ 确定主体内容重点
- [ ] 搭建框架后给领导过目，确认方向后再写正文

**脚手架三问检查表**：
- [ ] 为什么要写这个？（背景/意义清晰）
- [ ] 和之前的工作有什么关系？（有差异化分析）
- [ ] 领导想表达什么？（核心观点明确）

## B - 边界与误区（表格化）

### 适用边界
| 场景 | 是否适用 | 说明 |
|-----|---------|-----|
| 重要公文（讲话稿/汇报/请示） | 是 | 必须执行审题流程 |
| 常规性简单通知 | 否 | 可简化流程 |
| 重复性工作材料 | 否 | 已有参照，可简化 |

### 常见错误
| 错误类型 | 具体表现 | 正确做法 |
|---------|---------|---------|
| 闷头写 | 不沟通直接写，写完打回 | 先复述确认，连环追问 |
| 问宏观问题 | "这个材料怎么写" | "这个材料的重点是XX吗" |
| 思维定式 | 套用上一工作模板 | 先清空，再分析新任务的独特性 |
| 站位错误 | 写了下属的话或过细的话 | 用脚手架三问核对站位 |
| 只问一次 | 第一次回答只是表面原因 | 连环追问3个以上问题 |
