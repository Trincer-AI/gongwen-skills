## R - 原文定义

> 在我的理解中，标题有且仅有一个作用，就是为内容服务。标题来自内容，总结、提炼内容，如果通过标题看不出你想表达什么，那么，你写的标题就是一个'徒有其表'的标题。

**标题的唯一目的是服务内容**——好的标题应该能概括内容、让读者一眼看出文章主旨，而不是花里胡哨的排比对仗。

反面案例："快板风格"——标题花里胡哨，内容不知所云。常见于新人追求"亮眼标题"阶段，强行对仗导致形式大于内容。

## I - 自己的解释

**公文标题模式库**是执笔人积累的各种文种标题结构模板。但记住：**模式是工具，不是目的**。

标题的本质是**内容的精准概括**，而非形式的华丽展示。

常见标题模式分为三类：

### 一、结构型标题
- **三要素式**："发文机关 + 关于 + 事项 + 文种"（例：县教育局关于安全生产的通知）
- **正副题式**：正题概括主旨，副题说明范围（例：以高质量党建引领高质量发展——在庆祝建党100周年大会上的讲话）
- **单标题式**：直接概括主题（例：在全市疫情防控工作推进会上的讲话）

### 二、语言型标题
- **陈述式**：用完整句子（例：关于当前经济形势的报告）
- **祈使式**：用动词开头（例：关于切实做好防汛工作的紧急通知）
- **概括式**：用名词短语（例：全县乡村振兴工作汇报）

### 三、特色型标题
- **比喻式**："打好组合拳，跑出加速度"（用于工作推进类）
- **对仗式**："转作风、提效能、促发展"（用于作风建设类，需慎用，避免强行对仗）
- **数字式**："12345工作法"（用于创新做法类）

## A1 - 书中的案例

**遴选考试中最后写大标题**（pr-009）：
在遴选考试中，建议先写内容，最后再根据内容填写标题。因为刚拿到考卷时处于紧张状态，很难迅速想出精致的标题。在不停翻阅题干素材的过程中，可能看到让人有灵感的句子，进而提炼出标题结构。

**减法技巧——删除仍通顺**（fw-007）：
通读全文，看每一段的内容和小标题是否匹配。给句子做减法时，逐句反问"这个内容可不可以删除"——如果删除后，句子依然通顺、意思表述依然清晰，果断删。

## A2 - 触发场景

### 场景一：秘书触发
**情境**：领导让你写一份关于营商环境优化的会议通知
**你的行动**：
1. 检索本单位历史同类通知的标题模式
2. 按"三要素"结构组织：XX单位关于优化营商环境的通知
3. 检查标题是否清晰传达了会议的核心内容（时间/地点/主题）

### 场景二：思维顾问触发
**情境**：同事问"这个标题为什么看起来怪怪的"
**你的分析**：
- 检查三要素是否齐全（发文机关 + 事项 + 文种）
- 检查标题长度是否适中（一般不超过20字）
- 检查是否"徒有其表"——标题与内容是否匹配
- 检查是否有"快板风格"倾向（强行对仗但空洞）

### 场景三：角色扮演触发
**情境**：扮演上级领导审阅讲话稿
**你的模拟**：
- 看到标题"奏响绿色生产的'前奏曲'、技术升级的'协奏曲'..." → 评判：这是典型的"快板风格"，言之无物
- 看到标题"关于切实做好防汛工作的紧急通知" → 评判：简洁有力，一眼看出核心事项
- 建议：先让内容有料，再让标题有形

## E - 如何应用（质量检查点模式）

### 步骤一：建立个人标题模式库
**Input**: 无（从零开始积累）
**Output**: 按文种分类的标题模式库

**质量检查点**：
- [ ] 通知标题：结构/语言/常见句式都已收集
- [ ] 报告标题：结构/语言/常见句式都已收集
- [ ] 讲话稿标题：正副题结构/单标题结构都已收集
- [ ] 每周整理截图保存，标注好在何处

### 步骤二：先内容后标题
**Input**: 待写材料的完整内容
**Output**: 根据内容提炼的标题

**【用户确认】** 提炼标题前，请确认：
- [ ] 内容已写完整，不再大幅修改
- [ ] 已明确内容核心主旨
- [ ] 标题将基于内容而非先入为主的想法

**质量检查点**：
- [ ] 先把内容写完整，再提炼标题
- [ ] 标题能概括内容核心主旨
- [ ] 标题与内容绑定关系清晰

**常见问题修复**：
| 问题 | 检测方法 | 修复方案 |
|-----|---------|---------|
| 标题空洞 | 每个词在内容中找不到对应 | 补充具体内容支撑 |
| 标题过长 | 字数超过20字 | 精简到核心要素 |
| 标题跑题 | 换标题后内容需大改 | 重新提炼内容主旨 |

### 步骤三：用减法检验
**Input**: 写完的标题
**Output**: 通过减法检验的标题

**质量检查点**：
- [ ] 删除标题后，内容依然可理解（否→标题非必须）
- [ ] 换标题后，内容无需大改（否→绑定过深）
- [ ] 标题每个词在内容中有对应（否→标题空洞）

## B - 边界与误区（表格化）

### 适用边界
| 场景 | 是否适用 | 说明 |
|-----|---------|-----|
| 正式公文标题撰写 | 是 | 通知/报告/请示等 |
| 即兴发言/简短汇报 | 否 | 直接用讲话主题代替 |
| 文学性文章（散文/随笔） | 否 | 不在此范畴 |

### 常见错误
| 错误类型 | 具体表现 | 正确做法 |
|---------|---------|---------|
| 快板风格 | 标题花哨对仗，内容空洞 | 内容为王，标题服务于内容 |
| 本末倒置 | 先想标题再填内容 | 先写内容后提炼标题 |
| 三要素缺失 | 缺少发文机关或事项 | 按"发文机关+关于+事项+文种"检查 |
| 标题过长 | 超过20字，表述累赘 | 精简到核心要素 |
| 强行对仗 | 为了排比而排比 | 形式服务于内容，不做作 |

## F - 参考资源

### 标题模式检索
- **站内检索**：`site:gov.cn 关于X的通知`（获取上级部门规范表达）
- **历史检索**：本单位同类通知标题存档
- **同级参考**：兄弟单位公开文件标题

### 优质标题积累
- 建立"好标题"截图相册，按文种分类
- 标注每条标题好在何处（结构/用词/角度）
- 定期回顾，吸收内化为自己的语感
