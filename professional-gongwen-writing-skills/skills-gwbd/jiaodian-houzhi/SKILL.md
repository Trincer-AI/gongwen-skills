# 工作总结焦点后置框架法

**R（Reading）— 原文引用**
> 焦点后置：从工作面向的课题出发，以成效为焦点，标题采用两段式，后段是成效。

**I（Interpretation）— 理解重写**
焦点后置框架是一种从工作面向的课题出发、以成效为焦点的框架搭建方法。

核心是找到工作对各个对象产生的影响，以成效为核心组织材料。标题采用两段式，后段是成效。

成效分类维度：
1. **对上级** — 贡献了什么、完成了什么
2. **对服务对象** — 解决了什么问题、带来了什么改变
3. **对内部** — 提升了什么能力、完善了什么机制
4. **对外部/行业** — 产生了什么影响、形成了什么示范

这套框架适合：工作成果可以按影响对象分类、需要突出"解决了什么问题"、领导关注"干得怎么样"的结果。

**A1（past case）**
某县乡村振兴工作总结：

**旧框架**（按事项分）：
- 产业发展
- 生态保护
- 文化振兴
- 组织建设

**新框架**（焦点后置）：
```
一、产业兴旺开新局，集体经济收入增长30%
二、生态宜居展新颜，农村人居环境整治覆盖率达95%
三、乡风文明谱新篇，新建文化礼堂12个
四、治理有效创新路，"村民说事"制度实现行政村全覆盖
五、生活富裕迈新步，农村居民人均可支配收入增长8%
```

效果：以成效为焦点，每个板块都是"某方面取得某成效"的结构，成效可量化、对比清晰。

**A2（future trigger）**
你要写一份某市"文明城市创建"工作总结，需要突出创建工作对城市面貌、市民素质、城市形象的影响。如何用焦点后置框架组织？

**E（Execution）— 执行步骤**
1. **定课题**：明确这份总结面向的是什么课题（如"文明城市创建"）
2. **找成效**：分析工作对不同对象产生的影响（对上级/对群众/对城市/对行业）
3. **分类别**：按影响对象分类，每个类别提炼一个成效
4. **写标题**：标题采用两段式，后段是成效（"XX取得新突破"）

**B（Boundary）— 边界与局限**
- **失效场景**：工作成效不明显、难以量化、影响对象不清晰
- **常见错误**：成效空洞（只有方向没有具体成果）、分类混乱（同一层级用了不同标准）
- **作者盲点**：过于追求成效的亮眼，忽视了真实性和可核查性
