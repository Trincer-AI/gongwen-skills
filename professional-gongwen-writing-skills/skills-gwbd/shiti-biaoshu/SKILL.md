# 实质性表述追问法

**R（Reading）— 原文引用**
> 从工具性表述向实质性表述迈进的写作逻辑：操作层面→功能层面→目的层面→战略层面

**I（Interpretation）— 理解重写**
实质性表述是破解"写了但领导说没写清楚"困境的思维方法。

工具性表述 vs 实质性表述：
- 工具性表述写的是操作层面（我们做了什么），视角是"我做了什么"
- 实质性表述追问四层面：
  - 操作层面：我做了什么（工具性表述的起点）
  - 功能层面：工具能直接实现什么（功能改变）
  - 目的层面：为什么要改变（核心诉求）
  - 战略层面：驱动什么演进（组织形态根本性演进）

**A1（past case）**
"在五个营业厅投放服务引导机器人"的四层追问：
- 操作层面（工具性）：投放服务引导机器人
- 功能层面（第一层实质）：推动营业场所智能化转型
- 目的层面（第二层实质）：实现标准服务流程无人化执行
- 战略层面（第三四层实质）：推动线下服务渠道从业务办理场所向客户体验与价值交互中心战略转型

**A2（future trigger）**
你要写"在社区建设养老服务中心"的工作，不能只写"建了什么"，需要追问实质。

**E（Execution）— 执行步骤**
1. **识别工具性表述**：找到写了"做了什么"的句子（工具性表述）
2. **追问功能层面**：这项工作能直接实现什么改变？
3. **追问目的层面**：这个改变背后的核心诉求是什么？
4. **追问战略层面**：这会驱动什么组织形态的根本性演进？
5. **选择合适层面写作**：根据场合和需要选择表述层面

**B（Boundary）— 边界与局限**
- **失效场景**：简单的事务性报告不需要层层追问
- **常见错误**：所有工作都追问到战略层面，导致表述过于宏大
- **作者盲点**：不是越深越好，要根据材料性质和领导需求选择层面
