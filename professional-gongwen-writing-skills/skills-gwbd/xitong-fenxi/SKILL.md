# 系统分析法

**R（Reading）— 原文引用**
> 系统分析法：从思维方式上分析写作对象与文本，走进决策者关注的问题，解决下不了笔的问题。

**I（Interpretation）— 理解重写**
系统分析法是一种从思维方式上解决"下不了笔"的方法。

核心不是从"我有什么素材"出发，而是从"决策者关心什么问题"出发。把写作对象放在系统中分析，找到关键节点和维度。

三步分析：
1. **分析写作对象** — 这个对象在系统中处于什么位置
2. **分析文本** — 这类文本的功能是什么、领导想从中看到什么
3. **走进决策者需求** — 决策者拿到这份材料会怎么用

同时配合三化技巧：
- **重新定义分析单元** — 从对象本身→对象所处的更大系统
- **模式命名** — 将关键做法按工作维度归纳并命名
- **案例刻度** — 主体具名+效果量化+变化可改

**A1（past case）**
某国企写"数字化转型"工作报告：

旧做法：罗列技术细节（上了什么系统、买了什么设备、招了什么人）

新做法（系统分析）：
1. **分析对象**：数字化转型在业务系统中处于支撑位置，影响业务流程和组织架构
2. **分析文本**：领导想看的是转型对业务的具体影响，不是技术细节
3. **走进需求**：领导关注的是转型在行业中的位置、对竞争格局的影响

结果：框架调整为"战略层（为什么要转）/执行层（怎么转）/支撑层（靠什么保障）"，领导看完就知道转型对业务的影响。

**A2（future trigger）**
你要帮某县写一份"基层治理创新"调研报告，素材包括：网格化管理、吹哨报到机制、数字政务应用、志愿服务队伍建设。如何用系统分析法组织？

**E（Execution）— 执行步骤**
1. **分析写作对象** — 基层治理创新在社会治理系统中处于什么位置，与哪些因素相关
2. **分析文本** — 调研报告的功能是提供决策依据，领导想看到创新对治理能力的提升
3. **走进决策者需求** — 领导想了解：创新解决了什么问题、在区域中的位置、可复制性如何
4. **重新定义分析单元** — 从"某县的创新措施"上移到"区域治理格局中的位置"
5. **提炼模式** — 对关键做法进行模式命名（如"绣花式治理"）
6. **建立刻度** — 主体具名、效果量化、变化可改

**B（Boundary）— 边界与局限**
- **失效场景**：简单的汇报材料不需要这么深入的结构化
- **常见错误**：罗列数据没有结构，或结构太多太杂
- **作者盲点**：调研报告的功夫在调研前就开始了——带着结构性视角去调研
