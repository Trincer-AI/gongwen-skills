# 五抽屉归纳法

**R（Reading）— 原文引用**
> 公文写作的一个基本方法叫做合并同类项。写出来的文章像流水账，是因为没有用到这个方法。

**I（Interpretation）— 理解重写**
五抽屉框架是归纳主要做法和成效的逻辑框架：

| 抽屉类型 | 适用场景 | 框架维度 |
|----------|----------|----------|
| 职能切分型 | 系统性改革 | 制度供给/技术赋能/生态调试/标准牵引 |
| 生命周期型 | 阶段性工作 | 事前规范/事中优化/事后完善 |
| 二元结构型 | 建设工程类 | 硬件支撑与软件赋能 |
| 价值维度型 | 重大战略 | 政治账/经济账/文化账/民生账/生态账 |
| 要素递进型 | 要素流通 | 健全载体→降低流动成本→拓宽渠道→提升质量 |

**本质**：不是简单的分类，而是按照一定逻辑标准将内容收纳进不同"抽屉"。

**A1（past case）**
北京营商环境建设用职能切分型抽屉归纳：
- 高标准开展全国首个经营主体 → 制度供给抽屉
- 数字赋能 → 技术赋能抽屉
- 强化公平竞争审查 → 生态调试抽屉
- 打造国际一流北京标准 → 标准牵引抽屉

**A2（future trigger）**
写一段关于"科技创新工作"的总结，需要分类归纳。

**E（Execution）— 执行步骤**
1. **判断工作类型**：是系统性改革还是阶段性工作？是建设工程还是战略任务？
2. **选择抽屉框架**：
   - 系统改革 → 职能切分
   - 分阶段推进 → 生命周期
   - 基建项目 → 二元结构
   - 重大战略 → 价值维度
   - 要素流通 → 要素递进
3. **将内容装抽屉**：每个细分点归入一个抽屉
4. **检查一致性**：确认分类标准一以贯之

**B（Boundary）— 边界与局限**
- **失效场景**：内容简单不需要分类时强行使用；文学创作不适用
- **常见错误**：分类标准不一致（如时而按部门时而按时间）；抽屉选择不适合工作类型
- **作者盲点**：过于依赖框架可能让总结显得机械；要根据实际内容选择合适的框架
