# 材料修改平面化法

**R（Reading）— 原文引用**
> 将逻辑的承载主体从文章的结构层次转移到语言的范畴关系上

**I（Interpretation）— 理解重写**
当领导要求"大幅压缩又新增内容"时，表面上是篇幅与内容的双重压力，真正困境是旧有材料结构中篇幅与逻辑完整性深度捆绑。

解法：将垂直论证结构（执笔人思维）转化为平面清单结构（领导者思维）。

执笔人思维：垂直论证，像建造金字塔，追求根基稳固、枝差分明，力量来自严密的推导过程，体重也正源于此

领导者思维：平面清单，像作战地图，追求要点突出、指令清晰，力量来自高度的概括和果断的宣誓，轻盈也正源于此

三步骤：
1. 剥离观点盒与论述肉
2. 搭建平行维度新框架（最关键）
3. 用指令体填充新框架

**A1（past case）**
领导要求将数字化转型讲话稿压缩一半并增加"激活经营主体活力"和"强化数据安全"内容：
- 垂直结构：一层总论点→二层分论点→三层论据（论证血肉）
- 平面化结构：核心是便民/关键是兴业/底线是安全/方法是统筹
- 并列维度替代递进推导

**A2（future trigger）**
你写了一篇关于"基层治理创新"的讲话稿，领导要求压缩30%并增加"数字赋能"内容。

**E（Execution）— 执行步骤**
1. **剥离观点盒**：找结论性判断性指令性的句子
2. **识别论述肉**：描述现象、分析原因、解释重要性的句子（优先舍弃）
3. **明确新增指令**：将领导要求的新增内容纳入观点盒
4. **搭建平行框架**：寻找统摄性概念，拟定并列的平行标题
5. **指令体填充**：直接陈述去连接词，合并同类集约表达

**B（Boundary）— 边界与局限**
- **失效场景**：说理性强的议论文不需要平面化
- **常见错误**：保留太多论述肉，导致平面化不彻底
- **作者盲点**：这是结构重构不是修补，需要魄力
