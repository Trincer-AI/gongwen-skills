# 公文规范性三避坑法

**R（Reading）— 原文引用**
> 公文规范性是基础性避坑技能，三大问题：附体行文、标题回行、主送机关。

**I（Interpretation）— 理解重写**
公文规范性有三个常见坑需要注意：

1. **附体行文**：事务性公文（方案/规划/办法）不能作为附件，必须独立成文或用通知作为载体
2. **标题回行**：标题需要回行时，词义必须完整、不以虚词打头、不拆分词组
3. **主送机关**：必须是具有独立法人资格的机关，不能是内设机构；简称需核实

这三点是公文写作的基础性避坑技能，违反会被秒退秒拒。

**A1（past case）**
某单位印发工作方案的错误做法：
```
XX市XX局关于印发《XX工作实施方案》的通知

各科室：

经研究，现将《XX工作实施方案》印发给你们，请认真贯彻落实。

附件：XX工作实施方案
```

问题：事务性公文作为附件，违反附体行文规范。

正确做法：
```
XX市XX局关于印发XX工作实施方案的通知

各科室、直属单位：

经研究，现将《XX工作实施方案》印发给你们，请认真贯彻落实。
```

或者直接用通知形式发布方案内容（方案作为正文，不是附件）。

---

**A2（future trigger）**
你要帮某单位起草一份《关于印发<XX工作管理办法>的通知》，需要注意哪些规范性要点？

**E（Execution）— 执行步骤**
1. **查附体**：事务性公文不能作附件；如需用通知发布，应把内容作为正文而非附件
2. **查回行**：标题回行时确保词义完整，不拆分专有名词，不以虚词（的/了/在）开头
3. **查主送**：主送机关应是具有独立法人资格的机关全称或规范化简称，不能是内设机构

**B（Boundary）— 边界与局限**
- **失效场景**：事务性公文的简报、信息稿等不需要这么严格
- **常见错误**：以通知形式发布时把方案放在附件里、标题回行拆分了专有名词、主送用了内设机构简称
- **作者盲点**：认为这些都是"小问题"，但公文规范性错误会导致材料被秒退
